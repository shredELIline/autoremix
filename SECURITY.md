# Security policy

## Supported versions

Only the latest published preview and the latest `main` revision are supported.

## Reporting

Do not open a public issue for a suspected vulnerability. Use GitHub's private
security advisory flow for this repository and include:

- affected revision and platform;
- reproduction steps;
- impact;
- any known mitigation.

Do not include signing material, tokens, private media, or proprietary model
weights in a report.

## Scope

Security defects include unsafe media parsing, path traversal, unbounded cache
growth, native memory corruption, permission misuse, secret disclosure, and
model-download integrity failures.
